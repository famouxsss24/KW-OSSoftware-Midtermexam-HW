# 영상 녹화용 설명서 - "나는 어떤 무기체계 개발자일까?"

> 한 문단씩 카메라 보면서 읽으면 되는 구조로 짰음.
> 각 기능마다 **"이 기능이 뭔지 → 어느 파일/함수를 봐야 하는지 → 왜 이렇게 짰는지"** 순서로 정리.

---

## 0. 프로젝트 한 줄 요약

"**나는 어떤 무기체계 개발자일까?**"

10문항짜리 퀴즈로 사용자를 **유도무기 / 전자전 / 사격통제(C4I) / 임베디드** 네 가지 방산 SW 개발자 유형 중 하나에 매칭해주는 Streamlit 웹앱.
내가 직접 한화에어로스페이스·한화시스템·LIG넥스원 같은 방산 개발자가 목표라, 학우들도 자기 성향에 맞는 길을 찾을 수 있게 만들어 봤다.

## 1. 파일 구성

| 파일 | 역할 |

| `app.py` | 앱 메인. 라우팅/UI/로직 전부 여기. |

## 2. 기능별 설명 (영상 순서대로 읽으면 되는 스크립트)

### (1) 로그인 + 회원가입 - `page_login()`, `page_signup()`

"먼저 로그인 기능입니다. 아이디와 비밀번호를 입력받아서 `users.json` 에 저장된 해시값과 비교합니다.
비밀번호는 **평문으로 절대 저장하지 않고**, `hashlib` 의 `sha256` 을 써서 해시한 값만 저장합니다."

```python
def hash_password(pw: str) -> str:
    return hashlib.sha256(pw.encode("utf-8")).hexdigest()
```

"회원가입 쪽은 **검증 조건 5가지** 를 `validate_signup` 이라는 함수에 모아뒀습니다."
- 빈 값 제출 방지
- 아이디 영문/숫자만 허용 (정규표현식 `^[a-zA-Z0-9]+$`)
- 비밀번호 최소 4자
- 비밀번호 확인란 일치
- 아이디 중복 체크

"조건을 만족하면 해시해서 파일에 저장합니다."

---

### (2) 캐싱 - `@st.cache_data`

"정적인 데이터는 매번 불러오면 낭비니까, Streamlit 의 `@st.cache_data` 데코레이터를 써서 한 번 로드한 뒤 재활용합니다."

```python
@st.cache_data
def get_questions():
    return QUESTIONS
```

"질문 리스트, 결과 정보, 그리고 배경 이미지 base64 인코딩 결과까지 전부 캐시해뒀습니다. 페이지 전환할 때마다 다시 읽지 않습니다."

---

### (3) 퀴즈 + 이전 문항 버튼 (B3) - `page_quiz()`

"퀴즈 페이지입니다. 단순 카운터로 하면 이전으로 못 돌아가니까, **답변을 리스트로 저장** 하는 구조로 바꿨습니다."

```python
st.session_state["answers"] = [None] * len(QUESTIONS)
st.session_state["q_index"] = 0
```

"`answers` 리스트 안에 각 문항에서 고른 옵션의 인덱스가 들어있고, `q_index` 로 지금 보고 있는 문항이 몇 번인지 관리합니다.
'이전 문항' 버튼을 누르면 현재 선택값을 저장한 다음 `q_index` 를 1 감소시키고, 다음 문항 버튼은 1 증가시킵니다.
마지막 문항에서 '결과 보기' 를 누르면 점수를 집계하고 결과 페이지로 넘어갑니다."

진행률은 `st.progress()` 로 표시하고, 첫 문항일 때는 이전 버튼이 `disabled` 됩니다.

---

### (4) 결과 화면 - 레이더 차트 (A1) - `build_radar()`

"결과 페이지 최상단에 **Plotly 레이더 차트** 가 뜹니다. 네 가지 유형에 각각 몇 표 받았는지를 한눈에 보여줍니다."

```python
fig = go.Figure()
fig.add_trace(go.Scatterpolar(
    r=values_closed,
    theta=labels_closed,
    fill="toself", ...
))
```

"레이더 차트는 마지막 점이 처음 점과 이어져야 닫히기 때문에, 값을 리스트 끝에 한 번 더 붙여서 `values_closed` 로 넘겨줍니다.
배경은 투명 처리해서 앱의 어두운 배경이 그대로 비치도록 했습니다."

---

### (5) 결과 화면 - 공부해야 할 과목 / 분야 (A2)

"단순히 '이런 개발자네' 하고 끝내면 아쉬우니까, **유형별로 실제 공부해야 할 과목 + 핵심 키워드** 를 같이 보여주게 했습니다."

예를 들어 유도무기 유형이면:
- 광운대 학부 과목: 제어공학, 신호 및 시스템, 선형대수학, 확률 및 랜덤신호, 수치해석
- 공부 분야: 칼만 필터, 유도 항법(INS/GPS), MATLAB·Simulink

"데이터는 `RESULTS` 딕셔너리 안에 `courses` 와 `fields` 로 나눠 넣었습니다. 후배들이 이걸 보고 어느 과목부터 파면 되겠다는 감을 잡을 수 있게요."

---

### (6) 전체 사용자 통계 (C2) - `show_stats_chart()`, `stats.json`

"자기 결과만 보는 게 아니라 **다른 사람들은 어떻게 나왔는지** 도 보이면 재밌겠다 싶어서, 누적 응답 분포를 막대 그래프로 띄웠습니다."

```python
def add_stat(result_type: str):
    stats = load_stats()
    stats[result_type] = stats.get(result_type, 0) + 1
```

"퀴즈가 끝나면 `stats.json` 에 유형별 카운트를 누적 저장합니다. 결과 페이지에서 이걸 읽어서 Plotly 막대 그래프로 그리고, **내 결과 색깔을 다른 막대와 다르게** 칠해서 강조했습니다."

---

### (7) 배경 이미지 + 전역 스타일 (B2) - `inject_global_css()`, `get_background_b64()`

"배경은 **탱크 라인아트** 이미지를 깔고 싶었습니다. 그냥 `st.image` 쓰면 본문이랑 겹쳐서 안 되니까, CSS 를 직접 주입하는 방식을 썼습니다."

```python
@st.cache_data
def get_background_b64():
    if not os.path.exists(BG_IMAGE_PATH):
        return None
    with open(BG_IMAGE_PATH, "rb") as f:
        return base64.b64encode(f.read()).decode("utf-8")
```

"이미지를 base64 로 인코딩해서 CSS `url()` 안에 바로 박아넣었습니다. 따로 서버 경로를 잡아줄 필요가 없어서 단일 파일로 깔끔하게 돌아갑니다."

"그 위에 반투명 검은색 레이어(`rgba(0,0,0,0.55)`) 를 덮어서 가독성을 확보했고, 본문 카드(`.block-container`)에는 살짝 올리브톤 테두리를 줬습니다."

"만약 `data/background.png` 파일이 없으면, **fallback 으로 짙은 군용 그라데이션** 이 깔리도록 처리해서 파일 없어도 앱이 안 깨집니다."

---

### (8) 라우팅 (페이지 전환) - `if/elif` + `goto()`

"Streamlit 은 원래 한 화면이라, 여러 페이지처럼 보이게 하려면 상태 관리가 필요합니다. **세션 상태에 `page` 값** 을 두고, `main()` 함수 안에서 `if/elif` 로 분기해서 해당 페이지 함수를 호출하는 방식으로 구현했습니다."

```python
def main():
    init_session()
    inject_global_css()
    page = st.session_state["page"]
    if page == "welcome":
        page_welcome()
    elif page == "login":
        page_login()
    elif page == "signup":
        page_signup()
    elif page == "quiz":
        page_quiz()
    elif page == "result":
        page_result()
```

```python
def goto(page_name):
    st.session_state["page"] = page_name
    st.rerun()
```

"처음엔 PAGES 딕셔너리에 함수를 매핑해서 dispatch 하는 패턴을 썼는데, 솔직히 너무 헷갈려서 그냥 if/elif 로 풀어버렸습니다. 페이지가 5개밖에 안 되니까 이게 더 직관적이에요."

---

### (9) 개인화 / 차별화 포인트

"과제 요구사항 중에 **개인별 차별성** 이 있었습니다. 제 진로(방산 SW 개발자) 와 성향을 그대로 반영했습니다."

- **주제**: 일반적인 MBTI 가 아니라 방산 분야 4가지 직무로 좁힘.
- **문항**: 버그 발생 시 어디를 먼저 보는지, MCU 레지스터/주파수 스펙트럼/프로토콜 로그 같이 **업계에서 실제 쓰는 용어** 로 차별화.
- **결과 메시지**: 단순 "당신은 OO 유형!" 이 아니라, **성향 / 역할 / 강점 / 공부 과목 / 실제 취업 가능한 회사** 까지.
- **화면 구성**: 탱크 라인아트 배경 + 카키/올리브 톤 색상으로 방산 테마 통일.
- **진로 추천**: 한화에어로스페이스, 한화시스템, LIG넥스원 같은 실제 기업명을 넣음.

## 3. 실행 순서 (녹화할 때)

```bash
# 1) 가상환경 활성화
.\.venv\Scripts\Activate.ps1

# 2) 의존성 설치
pip install -r requirements.txt

# 3) 실행
streamlit run app.py
```

브라우저에서:
1. 첫 화면 - 타이틀 + 소개 확인
2. 회원가입 → 검증 메시지들 보여주기 (빈 값 / 4자 미만 / 중복 / 영문+숫자 외 / 비번 불일치)
3. 로그인
4. 퀴즈 - 3번째 문항쯤에서 **'이전 문항'** 눌러서 답 수정 가능한 거 보여주기
5. 결과 - 레이더 차트 / 공부 과목 / 진로 / 전체 통계 차례로 스크롤
6. 다시 풀어보기 → 다른 답 고르면 결과가 바뀌는 거 보여주기
7. 전체 통계 차트에서 **내 결과 색이 바뀌어 강조** 되는 것 확인

## 4. 파일 교체 체크리스트

지금까지 버전별로 쌓아둔 파일들을 실제 파일로 교체해야 돼:

1. `app_v6.py` → `app.py` 로 이름 바꾸기 (기존 `app.py` 백업 후 덮어쓰기)
2. `requirements_v2.txt` → `requirements.txt` 로 이름 바꾸기
3. 탱크 이미지를 `data\background.png` 경로에 저장
4. PowerShell 에서 정리용 명령:

```powershell
# 기존 app.py 백업하고 v6 으로 교체
Move-Item app.py app_v0_backup.py -Force
Move-Item app_v6.py app.py

# 중간 버전들도 정리 (영상 찍기 전에 깔끔하게)
Remove-Item app_v3.py -ErrorAction SilentlyContinue
Remove-Item app_v4.py -ErrorAction SilentlyContinue
Remove-Item app_v5.py -ErrorAction SilentlyContinue

# requirements 교체
Move-Item requirements.txt requirements_old.txt -Force
Move-Item requirements_v2.txt requirements.txt

# 테스트로 만들었던 쓰레기 파일 삭제
Remove-Item test_fresh_file.txt -ErrorAction SilentlyContinue
Remove-Item quiz_data.json -ErrorAction SilentlyContinue  # 루트에 생긴 18바이트짜리
```

5. plotly 설치:

```powershell
pip install plotly
```

6. 최종 확인:

```powershell
streamlit run app.py
```
